1.windows编程基础

1.应用程序分类

1.控制台程序
        扩展名exe，Dos程序，本身没有窗口，借用windows的Dos窗口执行，
                自身执行后进入内存，有入口函数
            
2.窗口程序
        扩展名exe，有自己的窗口，可以与用户交互，
                自身执行后进入内存，有入口函数
3.库程序
        1）静态库程序：扩展名lib，存放代码、数据程序，
                被可执行程序引用时，直接将内容复制到可执行程序中，本身不会进入内存，没有入口函数
        2）动态库程序：扩展名dll，存放代码、数据程序，
                被可执行程序引用时，本身会进入内存，有入口函数

xxxxxxxxxx

4.字符编码

1.字符编码集

xxxxxxxxxx
1.ASC编码：一个字符用7为二进制表示，含有英文字母大小写、数字、控制字符，共128个
2.ASCII编码：一个字符用8位表示，前128个不变，后128个用id区别，（德国id就后128个全是德国字母），共256个
3.DBCS：单双字节混合编码，英文用一个字节、中日韩等用两个字节编码，有缺陷，没有统一标准（一个汉字可能识别为两个英文字母）
4.UNICODE：万国码，所有国家的字符都有唯一编码
            utf-8，一般在linux下，
            utf-16，一般在windows下，所有字符都用16位表示（英文字母前面1位是0）
            utf-32

11.消息队列

xxxxxxxxxx
消息队列是存放消息的队列，只有GetMessage会在消息队列中抓消息，且GetMessage只能在消息队列中抓消息
消息在队列中先进先出
所有窗口程序都有消息队列




消息队列的分类：
        系统消息队列：
                操作系统维护的消息队列，存放系统产生的消息，例如鼠标、键盘，电脑启动就有，消息量大
        程序消息队列：
                属于每一应用程序（线程）的消息队列，有应用程序（线程）维护，程序启动才有
                
                /*
                    1.每个程序（线程）都有自己的GetMessage，每个GetMessage只能到自己的线程中抓消息
                    2.所有消息先进入系统消息队列，然后每隔一段时间，系统消息队列里的消息会被转发到各自进程的消息队列中
                    3.PostMessage把消息发到系统消息队列，然后进入程序消息队列，能被GetMessage抓到
                            SendMessage没有把消息发送到系统消息队列和程序消息队列，不能被GetMessage抓到
                    */

12.消息和消息队列的关系

xxxxxxxxxx
1.当鼠标、键盘产生消息时，会将消息存放到系统消息队列
2.系统根据存放的消息，找到对应的程序消息队列
3.将消息投递到程序的消息队列中




根据消息和消息队列之间的使用关系，消息分为
        队列消息：
                消息的发送、获取都是通过消息队列完成，（消息要进入队列）
                常见队列消息：WM_PAINT、键盘、鼠标、定时器（这些消息也可以不进队列，成为非对列消息）
                            WM_QUIT必须进队列
        非队列消息：
                消息的发送、获取，直接调用窗口消息处理函数处理完成，（消息不进队列）
                常见的非队列消息：SendMessage发送的消息就是非队列消息，不进队列，直接等待窗口处理函数处理和返回
                                WM_CREATE（必须不能进队列）、WM_SIZE等
                /*
                        消息本身没有队列消息、非队列消息之分，让其进入队列，就是队列消息，不让其进入队列，就是非队列消息
                */

13.深谈GetMessage

xxxxxxxxxx
1.GetMessage首先在本程序(线程)的消息队列中查找消息，若果有消息，检测消息是否满足指定条件（HWND，ID范围），不满足就不取，满足就从队列中取出消息返回
2.如果程序（线程）消息队列没有消息，向系统消息队列获取本程序的消息。如果系统队列的当前消息属于本进程，系统会将消息转发到程序消息队列中
3.如果系统消息队列也没有消息，检查当前进程的所有窗口的需要重新绘制的区域，如果发现有需要重新绘制的区域，产生WM_PAINT消息，并返回消息处理
4.如果没有重新绘制的区域，检查定时器，如果有定时器到设定时间，就产生WM_TIMER消息返回处理
5.如果没有到时得定时器，就整理程序的资源、内存等
6.GetMessage会继续等候下一条消息，而PeekMessage会返回FALSE，交出程序的控制权
            注意：GetMessage如果获取到WM_QUIT，函数会返回FALSE
    /*
    GetMessage：找不到，就要，要不到造，就造不了，造不了，就做零活，总后阻塞
    */

4.动态库

1.动态库的特点

xxxxxxxxxx
1.动态库的特点：
            1）运行时独立存在
            2）源码不会链接到执行程序
            3）使用时加载（依附于别的程序而运行）
2.j与静态库的比较
            1）静态库是将代码嵌入到使用程序中，多个程序使用时，会有多份代码，代码体积增大，而动态库只需要一份，其他程序通过函数地址使用，代码体积小
            2）静态库发生变化后，新的代码需要从新链接到执行程序中，动态库不需要（只要函数地址没变（即函数名）），因此静态库逐步淘汰

2.动态库创建

xxxxxxxxxx
1.创建动态库项目
2.添加库程序
3.库程序导出：提供给使用者库中的函数等信息
        方法1：声明导出：把_declspec(dllexport)写在函数定义前面
            
        方法2：项目中增加模块定义文件.def（与dll同名）     //（dll文件中函数名不换）
                例如：LIBRARY  文件名     //库
                     EXPORTS               //库导出表
                        函数名1    @1        //导出的函数
                        函数名2    @2        //导出的函数
    
    
                /*  注意：函数导出后，在动态编译连接后，也会有lib文件，是作为动态库函数映射使用，与静态库完全不同
                            导出的是函数地址（C++换了名的函数名）到.dll的文件头中（标号、函数名、地址），dll正文存的是函数源码，dll（文件头+正文）
                            对应lib存：函数名字和对应标号，还有对应dll文件名
                            */
                
    /*  C语言和C++动态库制作一样  */
    
    
4.使用
    1.隐式链接（操作系统负责动态库执行）
            1）头文件和声明函数原型：可以再函数原型的声明前，增加_declspec(dllimport) //注意此处是：dllimport
            2）导入动态库的lib文件       //注意：规定的只能到lib文件中抓编号+dll文件名，（dll文件太大），根据编号到dll文件找函数
            3）在程序中使用函数
            4）隐式链接情况下，dll文件可以存放的路径：
                        （1）与执行文件同一目录    /*只建议放在这里，其他地方要不是公共目录，要不就是可变的目录*/
                        （2）当前工作目录
                        （3）Windows目录
                        （4）Windows/System32目录
                        （5）Windows/System目录
                        （6）环境变量PATH指定目录     
    
    2.显示链接（程序员自己负责使动态库执行）

            2）加载动态库

5.线程

1.windows线程开发

xxxxxxxxxx
windows在客户端弱进线程开发，多用线程开发
    
1.windows线程是可执行代码的实例。系统以线程为单位调度程序。一个程序可以有许多线程，实现多任务处理，主线程只有一个。
2.windows线程特点：
        1）线程都有1个ID
        2）每个线程都有自己的内存栈，共享进程级数据（例如堆、代码段、数据段、BSS段（除了栈之外的））
        3）同一个进程中的多个线程使用同一个地址空间（进程只是分类存，现成才是代码的实例，cpu对应执行哪个线程）
3.线程的调度：
    操作系统将CPU的执行时间划分成时间片（每个时间片的时间很短），依据时间片执行不同线程
        线程轮询：线程A--->线程B--->线程...
