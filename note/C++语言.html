5.名字空间

xxxxxxxxxx
1.介绍：
名字空间（命名空间），表示一个作用域
定义在名字空间中的实体称为名字空间的成员
名字空间的作用：
        1）避免名字冲突        //同一空间下，不能重名
        2）划分逻辑单元


    
2.定义名字空间：
    使用namespace关键字定义名字空间
    名字空间成员可以是全局变量、函数、类型、名字空间
    
    namespace 名字空间名{
    int num=0;      //全局变量
    void func(...){...}     //函数
    struct Student{...};    //结构体
    namespace 名字空间名{}       //嵌套定义的名字空间
    
}




3.名字空间的使用：
    名字空间成员不能直接使用，主要有三种方法访问
    1）方法1：作用域限定操作符"::"
            名字空间名::成员
    
    2）方法2：名字空间指令
            using namespace 名字空间名;  //引入该空间所有成员，该指令后可以直接访问所有成员
            成员;
    
    3）名字空间声明：
        using 名字空间名::要访问的成员     //将特定成员引入，该指令后可直接访问该成员
                
                /*  
                    注意：1.使用方法2、3，若引入的空间含有相同成员，还是会冲突，可以用方法1
                         2.自定义名字空间一般写在main函数之前（其它模块写在最外面）
                         3.std是c++标准名字空间，可以访问预制的成员
                         4.一般情况下直接引入std（using namespace std;），其它空间用作用域限定操作符::
                    */
     
        
    4.无名名字空间：
        没有放在任何名字空间中的成员将被放入无名名字空间
            namespace {无名空间成员};     //加了namespace{}只能在当前文件访问，不加就是全局变量（其他文件也可以访问）
            ::无名空间成员        //访问
                
                
    5.名字空间嵌套：
         namespace ns1{
                namespace ns2{int num=100;}
            }
         
        cout<<ns1::ns2::num<<endl;

1.函数重载

xxxxxxxxxx
1.同一作用域下，函数名相同，参数列表不同（参数类型、参数个数、类型顺序），将构成重载关系（可以有很多个）   
    void  func(void){...}
    void  func(int i){...}
    void  func(int i,double d){...}
            //注意：函数重载的函数名一样，但地址不一样




2.函数重载与函数返回值无关，与形参变量名也无关        //函数声明时，可以不写形参变量名，只写类型


    
3.函数解析
        调用重载关系的函数时，编译器将根据实参参与形参的类型匹配情况，选择一个确定的重载版本
    
        void func(void){...}
        void func(int i){...}


        int mian(void){
            func();     //匹配第1个func
            func(123);  //匹配第2个func
        }




4.注意：
    void func(int num1,int num2);
    void  func(int num1,double num2);
    
    func(1,2);      //2个都能匹配，但第1个类型更符合，会执行第1个
    func(1,2.23);   //2.23默认是double，两个都不匹配，会报错，需要转换类型
    func(1,2.23f);      //匹配第2个
                    //重载会根据参数类型最匹配的一个执行


    //函数指针调用重载函数，会根据函数指针的类型，（函数指针类型在赋值时确定）  
    void (*pfunc)(int,float)=func;      //函数指针，指向int,float的重载版本，即第2个
    pfunc(10,20);       //调用第2个，20会隐式转化为float型      




5.匹配优先级
    一般规则：完全匹配>常来转换>升级转换>降级转换>省略号匹配      //由编译器决定
    
    //例1：
            //char-->int：升级转换
        void bar(int i){
            cout<<"bar(1)"<<endl;
        }


            //char-->const char：常量转换
        void bar(const char c){
            cout<<"bar(2)"<<endl;
        }


        int main(){
            char c='A';
            bar(c);     //会选择第2个，因为char-->int内存占用会变大，变成const char更接近


            return 0;
        }


    //例2：
        //short-->char：降级转换，能存的值数据范围会变小
        void  foo(char c){
            cout<<"foo(1)"<<endl;
        }
        
        //short-->int：升级转换，会占用更多的内存
        void  foo(int i){
            cout<<"foo(2)"<<endl;
        }
        
        int main(){
            short s=100;
            foo(s);     //默认会选择第2个版本，升级转换，防止数据丢失
        }
        


    //例3:
        //省略号匹配是最差的，最后匹配
        void hum(int i,...){        //,...表示可以接收任意个数任意类型的实参
                cout<<"hum(1)"<<endl;
        }
    
        void hum(int i,int j){
                cout<<"hum(2)"<<endl;
        }


        int main(){
            hum(10,1.23);   //会调用第2个（语法规定），第2个会进行降级转换，第1个,...参数具有不确定性（不会使用）
            return 0;
        }

1.语法
        名字一定是“~类名”
        没有返回类型，也没有参数(不能重载)      //一个类只有一个析构函数，不写送一个默认的，自动被调用
        主要负责清理对象生命周期的动态资源       
                class类名{
                    public:
                    ~类名(void) { ...}//析构函数
                    };


2.和构造函数函数相反，对象销毁时其析构函数将被自动调用
        栈对象，离开其作用域时析构函数自动被调用        //在创建对象的作用域的右大括号时调用
        堆对象，执行delete操作时析构函数自动被调用        
    
3.析构函数并不局限在释放资源上，它可以执行任何类的设计者希望在最后一次使用对象之后执行的动作
    
4.通常情况下，若对象在其生命周期的最终时刻，并不持有任何动态分配的资源，可以不定义析构函数.


5.如果类没有显式定义析构函数，那么编译器会为其提供一个缺省析构函数，缺省析构的函数体为空，在空析构函数体执行完毕后，类中的成员会被自动销毁
        对基本类型成员变量，什么也不做
        对类类型成员变量(成员子对象)，将会自动调用相应类的析构函数
                //自己写析构函数，需要自己删除对象


6.类中成员的销毁过程依赖于成员类型，销毁类类型成员变量需要执行它自己的析构函数，而内置类型没有析构函数，因此销毁内置类型什么也不需要做

1.浅拷贝
        //拷贝指针地址
        类中的缺省拷贝构造函数，对指针形式的成员变量按字节复制（复制的是地址，导致指向同一个），而不会复制指针所指向的内容，这种拷贝方式被称为浅拷贝。
        浅拷贝会导致不同对象之间的数据共享，对象在销毁时还可能会出现“double free”的异常

                String(const String& that) {

                    strcpy(m_str, that.m_str);
                }

                String& operator=(const String& that) {
                    if (this != &that) {
                        delete[] m_str; //释放原来的内存（原来的可能不够）
                        m_str = new char[strlen(that.m_str) + 1];//开辟新内存
                                                        //此处也可以先开新内存，再释放旧内存（如果开辟新内存失败，可以使用旧的）
                        strcpy(m_str, that.m_str);
                    }
                    return *this;
                }


                //向下访问的接口函数
                const char* c_str(void)const {
                    return m_str;
                }


            private:
                char* m_str;
            };

                               //拷贝赋值(深拷贝)

                    if (this != &that) {
                            String tmp(that);   //使用自定义深拷贝构造函数开辟内存
                            swap(m_str,tmp.m_str);      //交换后，tmp被自动释放

7.总结
        1.无论是拷贝构造还是拷贝赋值，其缺省实现都有可能是浅拷贝操作，因此应尽量避免使用指针型成员变量
        2.尽量通过引用或指针向函数传递对象型参数，既可以降低参数传递的开销，也能减少拷贝构造的机会
        3.出于具体原因的考虑，确实无法实现完整意义上的拷贝构造和拷贝赋值，可以将它们私有化，以防误用
        4.如果为一个类提供了自定义的拷贝构造函数，就也要提供实现相同逻辑的拷贝赋值运算符函数

14.静态成员

xxxxxxxxxx
1.为什么需要静态成员
        1）同一个类可以创建多个对象，那么每个对象都分别有一份自己的成员，内存上各自独立，互不相干，但很多时候希望某个成员可以为所有对象共有，内存唯一，这样可以实现数据共享
        2）在传统的C语言中可以通过全局变量实现数据共享，在程序的任何位置都可以访问全局变量，从而实现数据共享，但是全局变量安全性得不到保证，随处都可以修改全局变量，不利于程序的维护，在               程序的更新迭代中一旦失误，后果不堪设想
        3）C++中希望通过一个被局限在类中使用的全局资源，来实现该类所有对象的数据共享，这样很好规避全局变量所带来的问题，这就要使用静态成员


2.静态成员变量
        被static修饰的成员变量即为静态成员变量，需要在类的外部定义和初始化，同一个类构造的多个对象的静态成员共享（即静态成员依附于类，只有一份）
                class 类名{
                    static 数据类型变量名;//声明
                    };
                    数据类型类名::变量名=初值;//定义和初始化
                                    //静态成员变量属于类，不属于对象


3.注意：
        1）普通的成员变量属于对象，而静态成员变量不属于对象
        2）普通的成员变量在构造函数中完成定义和初始化，静态成员变量需要在类的外部单独定义和初始化
        3）静态成员变量和全局变量类似，被存放在全局区，可以把静态成员变量理解为被很制在类中使用的全局变量
        4）静态成员变量也要受到类的访问控制限定符的约束
        5）在类的外部访问静态成员变量
                类名:静态成员变量;
                对象.静态成员变量;//可以为对象共享，但不属于对象




4.静态成员韩函数
        被static修饰的成员函数即为静态成员变量
            class 类名{
                    访问控制限定符:
                    static 返回类型 函数名(形参表){ ... }
                    };
                            //注︰静态成员函数可以直接定义在类的内部，也可以定义在类的外部，这一点和普通的成员函数没有区别。




5.注意：
        1）静态成员函数没有this指针，没有const属性，可以把静态函数理解为被限制在类中使用的全局函数
        2）静态成员函数中只能访问静态成员，但是在非静态成员函数中既可以访问静态成员也可以访问非静态成员
        3）静态成员函数和静态成员变量一样，也要受到类的访问控制限定符的约束
        4）在类的外部访问静态成员函数
                类名::静态成员函数(实参表);
                对象.静态成员函数(实参表);

4.饿汉式和懒汉式
    1）饿汉式：
        将单例类的唯一对象实例，通过一个静态成员变量来实现，加载进程时即完成创建，无论用或不用该单例对象都一直存在
                优点：代码实现简单、访问效率高·多线程访问安全
                缺点：单例对象一直存在，浪费内存以空间换时间




    2）懒汉式
        将单例类的唯一对象实例，通过一个静态指针变量来维护，加载进程时仅有一个指针变量还没有创建单例对象，等用时再创建，不用即销毁
            优点：     
                    随用随创建，节省内存
            缺点:
                    创建销毁的过程浪费时间，多线程中需注意线程同步问题
                    以时间换空间

3.成员函数操作符重载例子（运算类）
        //复数：x+yi                         
        class Complex {
        public:
            Complex(int r,int i):m_r(r),m_i(i){}

            const Complex operator+(const Complex& c)const {

4.公有继承特性
        1）子类对象会继承基类的属性的行为，任何时候子类对象都可以被当做基类类型的对象，通过子类对象可以直接访问基类中的成员，如同是基类对象在访问它们一样
        2）子类对象中包含基类的部分可以称为“基类子对象”
        3）向上造型(upcast)
                将子类类型的指针或引用转换为基类类型的指针或引用;这种操作性缩小的类型转换，在编译器看来是安全的，可以隐式转换。
        4）向下造型(downcast)
                将基类类型的指针或引用转换为子类类型的指针或引用;这种操作性放大的类型转换，在编译器看来是危险的，不能隐式转化，但是可以显式转换
                        //向上安全，用得多，向下编译器都认为不安全，要程序员根据实际判断，需要强制转换，用得少

                ii.基类中的私有成员子类也可以继承，但是会受到访问控制属性的限制，不能直接访问;但是可以让基类提供公有或保护的成员函数来间接访问
        6）子类隐藏基类的成员

3.函数重写(虚函数覆盖)、多态的概念
        1）如果将基类中的某个成员函数声明为虚函数，那么子类中与该函数具有相同原型的成员函数就也是虚函数，并且对基类中版本形成覆盖，即函数重写.
        2）如果子类提供了对基类虚函数有效的覆盖，那么通过指向子类对象的基类指针，或者通过引用子类对象基类引用，调用该虚函数，实际被执行将是子类中的覆盖版本，
                而不再是基类中原始版本，这种语法现象被称为多态.
        3）多态的重要意义在于，一般情况下，调用哪个类的成员函数是由调用者指针或引用本身的类型决定的，而当多态发生时，调用哪个类的成员函数则由调用者指针或引用的实际目标对象的类型决定。
        4）这样一来，源自同一种类型的同一种激励，竟然可以产生多种不同的响应，也就是对于同一个函数调用，能够表达出不同的形态，即为多态
    


4.函数重写的要求(虚函数覆盖的条件)
        1）只有类中的成员函数才能声明为虚函数，而全局函数、静态成员函数、构造函数都不能被声明为虚函数‘
        2）只有在基类中以virtual关键字声明的虚函数，才能作为虚函数被子类覆盖，而与子类中的virtual关键字无关
        3）虚函数在子类中的版本和基类中版本要具有相同的函数签名，即函数名、参数表、常属性一致//子类该函数名与父类相同，父类该函数参数表必须包含于子类该函数参数表中，且常属性一致
        4）如果基类虚函数返回基本类型的数据，那么子类中的版本必须返回相同类型的数据
        5）如果基类虚函数返回类类型指针(A*)或引用(A&)，那么允许子类中的版本返回其子类类型指针(B*)或引用(B&)


5.产生多态语法的条件
        //函数重载只是多态的必要条件，多态还需要以下条件
        1）多态的语法特性除了要满足函数重写的语法要求，还必须是通过指针或引用调用虚函数，才能表现出来
        2）调用虚函数的指针也可以是this指针，当使用子类对象调用基类中的成员函数时，该函数里面this指针将是一个指向子类对象的基类指针，
                再通过this去调用满足重写要求的虚函数同样可以表现多态的语法特性
    
6.纯虚函数（抽象方法）
        只如果一个虚函数仅表达抽象的行为，没有具体的功能，即只有声明没有定义，这样的虚函数被称为纯虚函数或抽象方法，其语法特性如下︰
                class类名{
                public:
                virtual 返回类型 函数名(形参表)=0;        //=0表名纯虚函数，写空括号也可以，只是为了方便阅读
                    };
        //所以，前面哪个绘图程序，Shape中虚函数draw没有实际功能，每次都是调用的它的子类重写函数，因此基类该函数可以写成纯虚函数（多态的需要），不写函数体（没用）


7.抽象类和纯抽象类
        1）如果类中包含了纯虚函数，那么这个类就是抽象类
        2）如果类中的所有成员函数都是纯虚函数则可以称为纯抽象类        //所有普通成员函数都是纯虚函数
        3）抽象类往往用来表示在对问题进行分析、设计的过程中所得出的抽象概念，是一系列看上去不同，但本质上相同的具体概念的抽象描述
        //注∶无论是直接定义，还是通过new运算符，抽象类永远不能实例化为对象


8.虚析构函数
        1）问题：
        基类的析构函数不会自动调用子类的析构函数，所以当delete一个指向子类对象的基类指针，实际被执行的仅是基类的析构函数，子类的析构函数不会被执行，有内存泄漏的风险
                class A{ 
                    ~A(void){基类的析构函数}
                    }
                class B:public A{ 
                    ~B(void){子类的析构函数}};
                
                    A* pa = new B;//pa即为指向子类对象的基类指针
                    delete pa;//仅会调用基类的析构函数，有内存泄漏风险.
        2）解决：
            在基类虚构函数前加 virtual，成为虚析构函数，子类析构函数自动成为虚函数，形成重写        //区别于成员函数重写（函数名相同），虚析构函数的函数名不同（因为类名不同）
            形成多态语法，删除时会调用子类析构函数，之后子类析构函数总会自动调用基类析构函数，释放资源

        2）函数的异常说明是一种承诺，表示该函数所抛出的异常类型不会超出说明的范围。如果不遵守承诺抛出了异常说明以外的类型，则无法被正常捕获，而会被系统检测到，导致进程终止.

                ii.空异常说明，throw()，表示不会抛出任何异常
