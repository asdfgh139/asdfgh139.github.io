2.Qt中文编码

xxxxxxxxxx
Qt跨平台、跨地域，选择了统一编码，解决乱码现象
1.统一编码：
        包含世界上说有语言文字的字符集，Unicode

                    即程序中如何表示这个编码，则UTF就解决了这个问题，分为:UTF-8，UTF_16，UTF-32
3.Qt内部编码：
         使用UTF-16
4.外部编码：
         多样的，不同的

5.信号和槽

1.概念

xxxxxxxxxx
1.信号和槽是Qt自行定义的一种通信机制，实现对象之间的信号交互
2.当用户或系统触发了一个动作，导致某个状态发生了改变，该控件就会发射一个信号（即调用其类中的一个特定的成员函数（信号）），同时还可以携带必要的参数
3.槽和普通的成员函数几乎没有太多区别，可以是公有的、保护的、私有的，可以被重载，被覆盖，参数任意，且可以像普通成员函数一样调用
4.普通函数和槽函数并不在其语法特性，而在功能。槽函数跟多体现对某种特定信号的处理，可以将槽和其他对象信号建立连接，这样当发射信号时，槽函数将被触发和执行，进而完成具体功能

2.信号的定义

xxxxxxxxxx
class  XX:public  QObject{
    Q_OBJECT            //QT的宏标记，使得moc（元对像编译器）识别，处理扩展语法
signals:
     void signal_func(...);     //信号函数
    
}
                //注意：信号函数只能写声明，不能写定义（也不用写）

3.槽的定义

xxxxxxxxxx
class  XX:public  QObject{
    Q_OBJECT            //QT的宏标记，使得moc（元对像编译器）识别，处理扩展语法
public slots:
     void slot_func(...);       //槽函数
    
}
                //注意：槽函数可以连接到某个信号上，当信号被发射时，槽函数将被触发和执行，另外槽函数也可以当做普通函数的成员函数直接调用
                            //理解：信号函数发射信号，槽函数接收信号并处理

4.信号和槽的链接

6.信号和槽函数参数

xxxxxxxxxx
1.信号河槽参数要一致：
        QObject::connect(A,SIGNAL(sigfun(int)),B,SLOT(slotfun(int)));       //ok
        QObject::connect(A,SIGNAL(sigfun(int)),B,SLOT(slotfun(int,int)));       //error
2.可以带有缺省参数：
        QObject::connect(A,SIGNAL(sigfun(int)),B,SLOT(slotfun(int,int=0)));     //ok
3.信号函数参数多余槽函数参数，多余参数将被忽略
        QObject::connect(A,SIGNAL(sigfun(int,int)),B,SLOT(slotfun(int)));       //ok
    
                                //信号函数的参数会传给槽函数，因此槽函数需要的参数不能多余信号函数给的，同时类型也要满足要求

7.信号和槽的多个连接

xxxxxxxxxx
1.一个信号可以连接到多个槽（一对多）
            QObject::connect(A,SIGNAL(sigfun(int)),B1,SLOT(slotfun1(int)));     //ok
            QObject::connect(A,SIGNAL(sigfun(int)),B2,SLOT(slotfun2(int)));     //ok
                                //槽的触发顺序是不确定的
2.多个信号连接到一个槽（多对一）
            QObject::connect(A1,SIGNAL(sigfun1(int)),B,SLOT(slotfun(int)));     //ok
            QObject::connect(A2,SIGNAL(sigfun2(int)),B,SLOT(slotfun(int)));     //ok
                                //哪个信号发射，槽函数都能触发
3.两个信号可以直接连接（信号串级）
                QObject::connect(A1,SIGNAL(sigfun1(int)),A2,SIGNAL(sigfun2(int)));      //ok
                                //用在不在同一作用域的信号传递，用得比较少

6.面向对象的Qt编程

1.概念

xxxxxxxxxx
1.完全不使用任何面向对象技术，而只利用Qt说提供的类创建对象，并调对象的接口以满足用户的需求是可能的，但这样构建的应用的功能必然十分有限
2.首先，Qt类保护成员中的诸多实现无法再类的外部被复用，Qt试图通过多态实现的很多机制，例如事件处理，完全无法使用
3.再次，Qt提供的信号和槽不可能满足用户的所有需求，自定义信号和槽需要面向对象技术
4.最后，Qt设计师，Qt创造器等工具链都可以在面向对象的方式下使用Qt，反其道而行之不会有好结果

    QLabel* m_label;        //显示"+"
    QPushButton* m_button;  //"="按钮

#endif // CALCULATOR_H










//Calculator.cpp
#include "Calculator.h"


//构造函数
Calculator::Calculator()
{
    //界面初始化
    setWindowTitle("计算器");      //设置窗口标题




    m_editX=new QLineEdit(this);        //第一个加数，this即为当前父窗口指针
    m_editX->setAlignment(Qt::AlignRight);  //内部文本对齐方式右对齐
    m_editX->setValidator(new QDoubleValidator(this));      //数字验证器，只能输入数字


    m_editY=new QLineEdit(this);        //第二个加数，this即为当前父窗口指针
    m_editY->setAlignment(Qt::AlignRight);  //内部文本对齐方式右对齐
    m_editY->setValidator(new QDoubleValidator(this));      //数字验证器，只能输入数字


    m_editZ=new QLineEdit(this);        //显示结果控件，this即为当前父窗口指针
    m_editZ->setAlignment(Qt::AlignRight);  //内部文本对齐方式右对齐
    m_editZ->setReadOnly(true);              //只读的


    m_label=new QLabel("+",this);           //"+"


    m_button=new QPushButton("=",this);         //计算按钮
    m_button->setEnabled(false);                //默认禁用按钮




    //布局器，不用单独设置每个控件位置大小
    QHBoxLayout *layout=new QHBoxLayout(this);
    //按水平方向把每个控件添加到布局器中，从左到右有序的
    layout->addWidget(m_editX);
    layout->addWidget(m_label);
    layout->addWidget(m_editY);
    layout->addWidget(m_button);
    layout->addWidget(m_editZ);
    //应用布局器
    setLayout(layout);




    //信号和槽连接
    //左右操作数改变时，发送信号textChanged()
    connect(m_editX,SIGNAL(textChanged(QString)),this,SLOT(enableButton(void)));
    connect(m_editY,SIGNAL(textChanged(QString)),this,SLOT(enableButton(void)));
    //点击按钮，发送信号clicked
    connect(m_button,SIGNAL(clicked(void)),this,SLOT(calClicked(void)));
}


//"="按钮使能
void Calculator::enableButton(void){
    bool bXok,bYok;
    //text()获取输入的文本内容QString，toDouble()将QString转换为double型，参数保存是否转换成功
    m_editX->text().toDouble(&bXok);
    m_editY->text().toDouble(&bYok);


    //全为真，使能，有假，失能
    m_button->setEnabled(bXok && bYok);
}


//计算结果
void Calculator::calClicked(void){
    double res=m_editX->text().toDouble()+m_editY->text().toDouble();
    QString str=QString::number(res);   //number()能将double转为QString


    m_editZ->setText(str);      //显示结果
}

9.事件

1.概念

xxxxxxxxxx
1.在Qt中，是以事件驱动UI工具集，包括信号和槽都依赖于Qt的事件处理机制
2.通常事件是由窗口系统或Qt自身产生的，用以响应所发生的各类事情，比如用户按下并释放了键盘或鼠标、窗口因缩放而需要重绘、定时器到期而应有所动作

2.处理事件的方法

xxxxxxxxxx
1.在Qt中，事件被封装成对象，所有的事件对象类型都继承自抽象类QEvent
2.当事件发生时，首先被调用的是QObject类中的虚函数event()，其参数（QEvent）标识了具体的事件类型
3.在Qt桌面应用开发中，QWidgets类覆盖了其基类中的event()虚函数，并根据具体事件定要具体的事件处理函数
        void QWidgets::mousePressEvent(QMouseEvent* e);     //鼠标按下事件
        void QWidgets::mouseReleaseEvent(QMouseEvent* e);       //鼠标抬起事件
        void QWidgets::mouseMoveEvent(QMouseEvent* e);      //鼠标移动事件
        void QWidgets::paintEvent(QPaintEvent* e);      //绘图事件

3.如何处理事件

xxxxxxxxxx
1.所有的事件处理函数都是虚函数，可以被QWidgets的子类覆盖，以提供针对不同窗口控件类型的事件处理，控件的使用者所关心的往往是定义什么样的槽处理什么样的信号，而控件的实现者则关心覆盖哪些事           件处理函数
2.如果程序员希望在窗口中自定义的处理事件，可以继承WWidgets或其子类，比如QDialog、QMainWindow，在在定义窗口子类中重写事件处理函数，当相应事件被触发时，会利用多态的语法机制，所执行的事           件处理函数将是子类中重写的版本，从而实现程序员先要的事件处理效果

4.绘图事件

xxxxxxxxxx
1.通过绘图事件，可以实现自定义的图像绘制，当有下列情况之一发生时，将触发窗口的绘图事件，即QWidgets类的paintEvent()虚函数会被调用：
        1）窗口类被创建出来后第一次显示
        2）窗口由隐藏状态转变为可见状态
        3）窗口由最小化状态转变为正常或最大化状态
        4）窗口的因尺寸大小的变化需要呈现更多内容
        5）QWidgets类的update()/repaint()成员函数被调用
2.如果希望在自己的窗口中显示某个图像，在QWidgets的窗口子类中可以重写绘图事件函数paintEvent，在其中可用QPainter（Qt二维图像引擎）实现指定的图像绘制、渲染

2.数据库

xxxxxxxxxx
1.数据库操作语言（SQL）:
        SQL是一种针对数据库的结构化查询语言（简单的指令），用于实现数据库查询和程序设计，常用于关系数据库系统，实现数据存取、查询、更新等操作

3.Sqlite数据库

xxxxxxxxxx
1.Sqlite是一个轻量级的开源数据库，代码完全公开不受版权限制，实现了自给自足的、无服务器、零配置的SQL数据引擎，也是最广泛使用的关系式数据库
2.Sqlite（www.sqlite.org）提供了Sqlite的已编译版本和源程序，目前支持如下平台：Linux、Mac OS X、Windows、Windows Phone 8、Windows Runtime

4.Sqlite工作原理

xxxxxxxxxx
1.SQLite引擎不是程序与之通信的独立进程，而是以静态库或共享库的方式链接到程序中，而不同于客户-服务器数据库连接方式，所以主要的使用方法是在编程语言内直接API调用，当前主流语言，如C/C++、        java、python等都提供了sqlite操作的API接口
2.Sqlite在消耗资源、实时性、和可操作性有着出色的表现，这个数据库（定义、表、索引和数据本身）都在属主主机上存储在一个单一的文件中，通常Sqlite的工作是开始一个事务的时候锁定整个数据文件而         实现

5.Sqlite主要特性

xxxxxxxxxx
1.Sqlite不需要一个单独的服务器进程或操作系统（无服务器的）
2.Sqlite可以不用配置，安装或管理十分简单
3.Sqlite数据库是存储在一个单一的跨平台的磁盘文件
4.Sqlite是非常小、轻量级的，完全配置时小于400kB，省略可选功能配置时小于250kB
5.Sqlite是自给自足的，开发者不需要任何外部的依赖
6.Sqlite支持SQL92标准的大多数查询语言的功能
7.Sqlite事务是完全兼容ACID（原则性、一致性、隔离性、持久性）的，允许从多个进程或线程安全访问

8.SQL语句

xxxxxxxxxx
1.创建数据表
        CREATE  TABLE  表名（列名1  类型[约束],列名2  类型[约束],...）;     //关键字小写也可以，约束可有可无
        常用类型：INT(整型)、TEXT(字符串)、REAL(浮点数)
         常用约束：PRIMARY KEY（主键）：表示该列数据唯一，可以加快数据访问
                  NOT NULL（非空）：该列数据不能为空
2.删除数据表（一般不用，需谨慎，数据也会消失）
            DROP  TABLE  数据表名字;
3.插入数据
            INSERT  INTO  表名(列名1,列名2,...) VALUES(数值1,,数值2,...);
            如果要为表中所有列添加值，并且顺序插入，则不需要指出列名：INSERT  INTO  表名 VALUES(数值1,,数值2,...);
            例子：INSERT INTO company (id,name,age,address,salary) VALUES(10, "xiaoming", 18, "beijing", 3000.75);
                                /*  插入字符串必须用单引号，否则报错    */
4.删除数据
        DELETE  FROM 表名  WHERE  条件表达式;
            //可以没有子句，但数据表的所有数据都将被删除
            //如果有多个条件，使用逻辑and，or连接
        实例:DELETE FROM company WHERE id=2000;       //=和==都可以，都是等于
            DELETE FROM company WHERE address='北京' and salary<1000;
5.修改数据
        UPDATE  表名  SET  列名1=新数值,列名1=新数值,... WHERE 条件表达式;
            //可以没有WHERE子句，但数据表所有将被修改
            //新数值可以是一个常数，也可以是表达式（变量）
            实例：UPDATE  company  SET  age=45 WHERE id=200;
                 UPDATE  company  SET  salary=salary+2000  WHERE age>=30 and age <=35;
5.查询数据
        SELECT  列名1,列名2,... FROM  表名;
        SELECT  列名1,列名2,... FROM  表名  WHERE 条件表达式;
        SELECT  列名1,列名2,... FROM  表名  WHERE 条件表达式 ORDER BY 列名 排序方式;
            //*表示查询所有列
            //注意ORDER BY和WHERE子句可以配合使用，也可以单独使用
            实例：SELECT * FROM company WHERE salary>10000 or salary <3500;
                 SELECT * FROM company ORDER BY id ASC;     //ASC从小到大，DESC从大到小
6.模糊查询
        SELECT  列名1,列名2,... FROM  表名  WHERE 列名  LIKE  模糊匹配条件;
            模糊匹配通配符：
                    百分号%：代表0个、1个或多个数字或字符
                    下划线_：代表一个单一的数字或字符
                SELECT * FROM company WHERE salary LIKE  "8___"         //三个下划线，工资8000多的人（注意"8___"匹配不到浮点数8000.0）
                SELECT * FROM company WHERE name LIKE  "张%"         //查询姓张的

3.TCP协议

xxxxxxxxxx
1.TCP（传输控制协议）是一个可靠的、基于字节流的，面向连接的传输层协议
2.在OSI定义的七层网络模型中，tcp属于第四层传输层所指的功能，在TCP/IP协议族中，TCP位于IP协议层之上，应用层之下的中间层
3.不同主机的应用层之间经常需要可靠的、像管道一样的连接，但是IP层不提供这样的流机制，TCP协议恰恰复合这一要求，特别适合连续传输数据，同时保证数据的安全性，广泛使用的HTTP协议就是基于TCP协             议实现的

    QTcpSocket* tcpClient= tcpServer.nextPendingConnection();
    //保存套接字到容器
    tcpClientList.append(tcpClient);
    //当客户端向服务端发送消息时，通过套接字发送信号：readyRead

//接收客户端消息槽函数
void ServerUi::onReadyRead(){
    //遍历容器，是哪个客户端发送的消息
    for(int i=0;i<tcpClientList.size();i++){
        if(tcpClientList.at(i)->bytesAvailable()){   //bytesAvailable()：获取当前套接字等待读取消息的字节数
                //读取消息并保存到缓冲区
            QByteArray buf=tcpClientList.at(i)->readAll();

            ui->listWidget->addItem(buf);
            ui->listWidget->scrollToBottom();   //消息滑动到最底部

        }
    }


}

    for(int i=0;i<tcpClientList.size();i++){
        tcpClientList.at(i)->write(buf);
    }
}




//定时器槽函数：定时删除断开的客户端套接字
void ServerUi::onTimeout(void){
    //遍历检查
    for(int i=0;i<tcpClientList.size();i++){
        if(tcpClientList.at(i)->state()==QAbstractSocket::UnconnectedState){
            //断开连接时删除该客户端的套接字
            tcpClientList.removeAt((i));
            i--;        //删了一个，调整一下i
        }
    }
